# M4.8a — Metadata-Driven Adapter Engine (Tool Factory)

**Status:** implemented — branch `feat/m4.8a-adapter-engine` (pending).
**Prereqs:** M4.1 `GuardedExecutor`/idempotency, M4.3 per-user OAuth (`CredentialResolver`), M2.2a RBAC gate.
**Companion docs:** [`CLAUDE.md`](../constitution/CLAUDE.md) §5/§6, [`ARCHITECTURE.md`](../architecture/ARCHITECTURE.md).

> Run Corridor `analyzePlan` **before** writing code and **again** on the diff.

## Context

Adding each new outbound tool today means a hand-written `BaseTool` subclass + provider shim (~80 lines
each). M4.8a introduces a **metadata-driven adapter engine**: a trusted, version-controlled JSON
`ToolSchema` is loaded at startup and turned into a registry-ready `BaseTool`, so new HTTP/JSON tools
become "add a reviewed schema." The engine is a **factory**, not a second security path — execution
still flows through the unchanged executor gate (RBAC re-check + `action_id` approval), `GuardedExecutor`
(idempotency), and the hash-chained `AuditLog`.

**Honest scope note.** atlas already centralizes RBAC/approval/audit/idempotency, so the engine's real
win is removing per-tool HTTP/payload boilerplate — not "centralizing security that was fragmented."
This phase proves the engine by **re-expressing one existing tool** (`slack_post_as_user`) as a schema
with no behavior change; no new outbound capability is added.

## Decisions (locked)

- **Risk tier stays code-anchored.** A schema may reference a `RiskTier` but defaults to `SEND`; it can
  never declare an auto-run `READ` unless the tool name is on a code-reviewed allowlist (empty today).
  Lowering a tool to auto-run therefore requires a code+review change, never a JSON edit.
- **Schemas are a trusted build artifact** under `src/atlas/tool_schemas/`, loaded at startup — never
  user/network/LLM-supplied, never hot-loaded.
- **SSRF-safe egress.** The endpoint host is validated against an allowlist at build time and re-checked
  by the transport at call time; the URL is from the schema, never from args.
- **Tokens server-resolved** via `CredentialResolver` (scoped to `(org_id, user_id)`); never from args/logs.
- **Declarative, non-eval transforms.** Payload/response mapping is a static field map (copy an arg or
  emit a constant) — no expression evaluation or templating. (JSONPath/nested mapping deferred until a
  connector needs it.)
- **Off by default** (`ATLAS_ADAPTER_ENGINE_ENABLED`), so the default graph and the eval gate are
  byte-for-byte M4.7.

## Changes

1. `src/atlas/adapter_engine.py` (new): `ToolSchema`/`ArgSpec`/`PayloadField`/`ResponseField` (frozen,
   `extra="forbid"`), `AdapterEngine.build_tool` / `build_tools_from_dir`, `load_schema`,
   `packaged_schema_dir`. Generates a Pydantic args model from the schema.
2. `src/atlas/tool_egress.py` (new): `Transport` ABC + `HttpxTransport` (sync, no retries) +
   `FakeTransport`; `assert_host_allowed` / `EgressNotAllowed`.
3. `src/atlas/tool_schemas/slack_post_as_user.json` (new): first schema (mirrors `SlackUserApiSender`).
4. `src/atlas/tools.py`: `ToolRegistry.register(..., replace=False)`.
5. `src/atlas/orchestration/graph.py`: `_apply_adapter_engine` registers schema tools (replacing
   hand-written twins) when the flag is on; no node/gate changes.
6. `src/atlas/config.py`: `adapter_engine_enabled`, `adapter_egress_allowlist`, `adapter_schema_dir`.

> **Path note:** the plan named `src/atlas/tools/…`; to avoid converting the `tools.py` module into a
> package (churn + cycle risk), the engine lives at `src/atlas/adapter_engine.py` and
> `src/atlas/tool_egress.py`. The chosen first tool is `slack_post_as_user` (not `slack_post`): the
> latter uses `slack_sdk` + a managed bot token and is a poor fit for an HTTP/JSON + `CredentialResolver`
> engine, while `slack_post_as_user` matches the engine's lifecycle exactly.

## Tests (`tests/test_adapter_engine.py`)

Equivalence: schema metadata == hand-written; identical `ProposedAction`; identical payload + result
shape (`{ts, channel, provider:"slack_user"}`); schema tool is approval-gated; generated args enforce
`min_length`/`max_length`. Security: schema declaring `read` rejected; default tier is `SEND`;
off-allowlist host rejected at build and at call time; missing resolver/transport fails closed;
`ok:false` raises; unknown/ambiguous schema fields rejected; `_apply_adapter_engine` swaps in the tool.

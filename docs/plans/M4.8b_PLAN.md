# M4.8b — Proxy-Bound Egress

**Status:** implemented.
**Prereqs:** M4.8a adapter engine + SSRF-hardened `HttpxTransport` + `_apply_adapter_engine`.
**Companion docs:** [`M4.8a_PLAN.md`](M4.8a_PLAN.md), [`CLAUDE.md`](../constitution/CLAUDE.md).

## Context

Enterprise deployments often require all outbound API traffic through a central forward proxy (Squid,
Envoy, corporate gateway). M4.8a intentionally deferred proxy mode because a misconfigured proxy is
itself an egress risk. M4.8b adds an **opt-in, isolated, tested** proxy transport while keeping the
existing direct (IP-pinned) path as the default.

## Decisions (locked)

| Decision | Choice |
|---|---|
| Default mode | **Direct** (`HttpxTransport`) when `ATLAS_ADAPTER_EGRESS_PROXY_URL` is blank |
| App-layer guard | **Unchanged** — every call validates destination URL against `EgressPolicy` before network I/O |
| Per-user OAuth | **Stays in app** — `CredentialResolver` → `Authorization: Bearer` on the destination API request |
| Env proxy hijack | **`trust_env=False` always** — never honor `HTTP_PROXY`/`HTTPS_PROXY` from the environment |
| IP pinning in proxy mode | **Skipped** — proxy reaches the target; app still enforces schema route contract |
| Scope | **Schema tools only** (adapter engine path); hand-written tools unchanged |

## Changes

1. `src/atlas/tool_egress.py`: `prepare_validated`, `ProxyTransport`, `make_adapter_transport`.
2. `src/atlas/config.py`: `ATLAS_ADAPTER_EGRESS_PROXY_*` settings + startup validation.
3. `src/atlas/orchestration/graph.py`: wire `make_adapter_transport` in `_apply_adapter_engine`.
4. `tests/test_tool_egress.py` + `tests/test_adapter_engine.py`: hermetic proxy-mode coverage.
5. `docs/guides/RUNBOOK.md`: operator guidance for proxy vs direct mode.

## Tests

- `make_adapter_transport` with blank proxy → `HttpxTransport`
- Proxy mode: `proxy=` kwarg, `trust_env=False`, Bearer on API request, optional `proxy_auth`
- Off-allowlist / wrong route → `EgressNotAllowed` before client call
- Redirect → `EgressNotAllowed`; proxy URL userinfo rejected at config/transport init
- `_apply_adapter_engine` with proxy URL → `ProxyTransport` on registered schema tools

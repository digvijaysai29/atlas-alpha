# M4.3 — Per-user OAuth + Credential Vault (HashiCorp Vault)

**Status:** implemented on `feat/m4.3-oauth-vault`.
**Prereqs:** M4.1 email + M4.2 Slack + M3.3 OIDC + Postgres audit.

## Outcome

Per-principal OAuth token storage in **HashiCorp Vault KV v2**, with gated tools that act as the authenticated user:

- `gmail_send` (`tool:gmail:send`)
- `calendar_create_event` (`tool:calendar:write`)
- `slack_post_as_user` (`tool:slack:post_as_user`)

Service-identity tools (`send_email`, `slack_post`) are unchanged.

## Key modules

| Module | Role |
|---|---|
| `governance/credentials.py` | `CredentialVault` ABC, `InMemoryCredentialVault`, `CredentialResolver` |
| `persistence/hashicorp_vault.py` | `HashiCorpCredentialVault` (hvac KV v2) |
| `integrations/oauth.py` | Authlib Google/Slack OAuth clients |
| `interface/oauth_routes.py` | Connect/callback/connections/revoke |
| `integrations/gmail.py` | Gmail API sender |
| `integrations/calendar.py` | Google Calendar client |
| `integrations/slack_user.py` | Slack user-token post |

## Config

See `.env.example` — `VAULT_ADDR`, `VAULT_TOKEN` (dev) or AppRole, Google/Slack OAuth client creds.

Local Vault: `docker compose up -d vault` (dev root token `dev-root-token`).

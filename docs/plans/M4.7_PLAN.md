## Context

`/chat` and `/approve` are **blocking, single-response** endpoints: the caller waits for the whole graph
(`planner → approval → executor → responder`) before getting one JSON body. M4.7 adds a **Server-Sent
Events** sibling, `POST /chat/stream`, so a turn streams its lifecycle — `open → node* →
(awaiting_approval | completed) → done` — over one connection, **reusing the existing OIDC auth, rate
limiting, and identity spine unchanged**. The blocking `/chat` is untouched (full back-compat).

**Scope:** v1 streams **lifecycle + interrupt + final-result** events (LangGraph `stream_mode="updates"`),
not per-token deltas — the responder emits a *deterministic* summary, not streamed LLM tokens. True
token streaming (LangGraph `stream_mode="messages"`, which consumes the provider stream *inside* the
graph, preserving nodes/interrupts/governance) is a future hook; raw-provider SSE passthrough is
explicitly rejected (it would bypass the checkpointer, approval interrupt, RBAC, and audit).

## Decisions (locked)

- **Additive `POST /chat/stream`**, `text/event-stream`. Approval decisions still go through the sync
  `/approve`; `/chat/stream` only *surfaces* the interrupt. `/approve/stream` is deferred (it would need
  `verify_thread_owner` before the body streams).
- **`sse-starlette` `EventSourceResponse` for the transport** (managed framing + keep-alive ping +
  disconnect handling), not a hand-rolled formatter. One new dependency; no new env/secret.
- **Sync graph, off the event loop.** The persistence stack is sync (psycopg pool + `PostgresSaver`),
  so `astream` is not used; the blocking `graph.stream(...)` iterator is advanced one `next()` at a time
  in a worker thread (`anyio.to_thread`), keeping the event loop free.
- **Auth/limit are dependencies → they run before the body.** `RequestPrincipal` (401) + `RateLimited`
  (429) resolve before the 200 streams; a rejected caller gets a normal JSON error, never a stream.
- **Events reuse the existing shaping helpers** (`_response_from_invoke` / `_response_from_snapshot`):
  RBAC-filtered `sources`, `confidence`, `pending_actions`, `action_results`. `node` events carry the
  node name only.

## Tests

- **`tests/test_interface_stream.py` (offline):** read-only turn → `completed`; send turn →
  `awaiting_approval` (no `completed`); `node` events are name-only (no action payload); **401 before
  any event** (OIDC app, no bearer); **429 before any event** (deny limiter); **mid-stream failure →
  generic `error` event** with no internal/exception leak, stream still closes with `done`.

## Security invariants (must hold)

1. **Authn/authz/limit before the body** — `RequestPrincipal` (401) + `RateLimited` (429) are
   dependencies; a rejected caller never receives a 200 event stream.
2. **No new IDOR** — `/chat/stream` creates a fresh caller-owned thread; no cross-thread access.
   `/approve/stream` is deferred precisely because it needs `verify_thread_owner` first.
3. **No leakage mid-stream** — errors emit a generic `error` event; exception/secret/KG detail is logged
   server-side only. `node` events carry node names, not action payloads.
4. **Same data contract** — events expose only `AgentResponse` fields, shaped by the existing
   RBAC-filtered helpers; risk tiers/pending actions come from state, never the model.
5. **Durable + fail-safe** — the checkpointer persists state; a dropped SSE connection loses no work
   (the thread remains readable via `/threads/{id}`).
6. **No new env/secret/serde type** — one new managed dependency (`sse-starlette`); transport-only models
   stay out of the `atlas_serde()` allowlist; the eval gate stays **1.00**.
